# PRD - 개발 블로그 검색(Dev Search)

Velog 개발 블로그의 글을 수집/정제하여 검색 가능한 형태로 제공한다.

## 1. 목적(Goals)
- 수집된 글을 빠르게 탐색하고, 키워드/태그 기반으로 원하는 글을 찾는다.
- DB 기반 기본 목록 + ES 기반 검색을 병행하여, ES 장애/미사용 환경에서도 최소 기능(목록/상세/간단 검색)이 유지된다.

## 2. 범위(Scope)
- 대상 데이터: Velog 글(수집기는 1회성 적재를 기본으로 함)
- 화면(UI)
  - 목록: 기본 목록 + 검색어 입력 + 결과 탐색(태그 기반 탐색 포함)
  - 상세: 본문/메타정보 확인, 원문 링크 이동
- 검색 데이터 소스
  - 기본 목록/상세는 DB 원본(`velog_post`) 기반
  - 검색어가 있을 때 Elasticsearch를 사용하여 검색 결과의 후보/순서를 결정하고, 최종 표시는 DB로 hydrate 한다.

## 3. 비범위(Non-goals)
- 실시간/주기적 수집(스케줄러 운영)
- 개인화 추천/랭킹 고도화
- 동의어/오타 교정/사전 기반 형태소 튜닝(user_dictionary) 고도화

## 4. 데이터 흐름
1. 수집기: Velog API를 통해 글을 가져와 DB(`velog_post`)에 적재
2. (선택) 색인: DB 글을 ES 인덱스에 색인하여 검색 성능/품질을 제공
3. 검색: 사용자가 검색어를 입력하면 ES에서 검색(후보 ID + 관련도/집계)을 수행하고, 화면 표시는 DB에서 ID로 조회하여 보여준다.

## 5. 검색 요구사항
### 5.1 검색 대상 필드
- `title`, `body`, `tags`를 검색 대상으로 한다.

### 5.2 페이징/정렬
- 페이징은 `page/size` 기반.
- ES 기반 검색일 때는 ES 결과 순서를 화면 결과에 반영한다(관련도 정렬 유지).
- 기본 목록(검색어 없음)은 DB 최신순 정렬을 기본으로 한다.

### 5.3 탐색 UX(ES 활용)
아래 항목은 “ES를 쓰는 의미”가 사용자에게 보이도록 UI에 함께 표출한다.
- `total hits`: 검색 결과 총 개수
- `took(ms)`: 검색 처리 시간(지표)
- `_score`: 결과별 관련도 점수(지표)
- `top tags`: 결과 집계 기반 상위 태그(태그 클릭 시 재검색)
- 검색어 자동완성: 입력 중 제목 기반 후보 제안

### 5.4 예외/장애 정책
- ES가 비활성화/장애인 경우:
  - 검색은 DB LIKE 기반으로 fallback 한다(최소 기능 유지)
  - 기본 목록/상세는 정상 동작해야 한다.
