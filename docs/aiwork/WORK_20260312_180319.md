# WORK_20260312_180319

## Summary
- Replaced sample/demo code with a DB-backed login flow (Spring Security + MyBatis).
- Added SHA-256 password verification (hex) and role mapping (`roles`, `user_roles`).
- Added login audit logging to `login_logs` for SUCCESS/FAIL with IP and User-Agent.
- Built a shared Thymeleaf layout with menu + content pages and moved CSS to static files.
- Updated `docs/` to reflect the current state and introduced an `aiwork` logging rule in `README.md`.

## Functional Behavior
- Login UI: `GET /login` (Thymeleaf)
- Login processing: `POST /login` (Spring Security form login)
- Success redirect: `/`
- Logout: `POST /logout` -> `/login?logout`
- Audit on every attempt: inserts into `login_logs` with `login_result` in `SUCCESS`/`FAIL`

## Database
Uses MySQL tables:
- `users` (`username`, `password`, `status`)
- `roles` (`role_name`)
- `user_roles` (mapping users to roles)
- `login_logs` (login attempt history)

Password storage:
- Stored as SHA-256 hex (example: MySQL `SHA2('admin', 256)`).
