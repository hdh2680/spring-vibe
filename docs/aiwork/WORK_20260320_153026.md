## 목표 (goal)
- 감정분석(사전 기반)을 위해 기존 4개 사전 파일을 하나의 TSV로 통합하고, 로직에서 빠르게 읽을 수 있도록 앱 시작 시 1회 로드 + 캐시 구조를 만든다.
- 감정 점수화는 n-gram 최장일치(max match)로 계산할 수 있게 기반 코드를 준비한다.

## 핵심 변경사항 (key changes)
- 사전 통합
  - `SentiWord_Dict.txt`(점수 포함) + `pos/neg/neutral` 리스트를 합쳐 `sentiment_lexicon.tsv` 생성.
  - 우선순위: 점수 포함(`knu_scored`) 항목이 최우선, 리스트(pos/neg/neutral)는 `+1/-1/0`으로 보강.
  - 충돌(예: pos/neg 양쪽에 존재)은 안전하게 0으로 중립 처리.
- 로더/캐시
  - `SentimentLexiconLoader`에서 `classpath:static/docs/sentiment_lexicon.tsv`를 최초 1회 로드하여 메모리에 캐시.
- 점수화
  - `SentimentAnalyzer`에서 토큰 리스트 기준 n-gram(2~max) 최장일치로 매칭해 점수를 합산.
