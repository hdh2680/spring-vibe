# 목표 (goal)
UTF-8 BOM 재발 방지를 위해 코딩 규칙 문서에 인코딩 규칙을 추가하고, 커밋 단계에서 BOM을 차단할 수 있는 스크립트/훅을 제공한다.

# 핵심 변경사항 (key changes)
- `/docs/CODING_RULE.md`에 "UTF-8 (no BOM)" 규칙과 BOM 체크 스크립트/훅 안내를 추가했다.
- BOM 체크 스크립트 `docs/scripts/check_utf8_bom.ps1`를 추가했다.
- staged 파일에 UTF-8 BOM이 있으면 커밋을 막는 Git pre-commit 훅 `.git/hooks/pre-commit`를 추가했다.

# 수정/추가한 파일 (touched files)
- `docs/CODING_RULE.md`
- `docs/scripts/check_utf8_bom.ps1`
- `.git/hooks/pre-commit`

# 실행/테스트한 명령 (run/test commands)
- `.\mvnw.cmd -q -DskipTests compile`

# 후속 할 일 (follow-ups)
- 기존에 BOM이 포함된 리포지토리 파일들(`docs/`, `src/main/resources/`)을 일괄 no-BOM으로 정리할지 결정.
