# 목표 (goal)
- 사전 기반 감정분석에서 중립(0점)으로 과도하게 떨어지는 현상을 완화하기 위해, **사전 매칭(hit)율을 올리는** 현실적인 개선을 로직에 반영한다.

# 핵심 변경사항 (key changes)
- `SentimentAnalyzer`에서 아래 휴리스틱을 적용해 사전 매칭(hit)율을 올리도록 개선했다.
  - 토큰 양끝 문장부호 제거(예: `좋다!` -> `좋다`)
  - unigram 매칭 실패 시에만 “후보 형태(candidates)”를 추가로 시도:
    - 인터넷 슬랭 강조 접두어 `개` 제거(예: `개좋다` -> `좋다`)
    - 자주 쓰는 조사/어미 일부를 뒤에서 제거한 형태(간이 stemming)
- 부정 처리:
  - 공백 기준 토큰이 `안/못/전혀/절대/별로/...`인 경우, **바로 다음 감정어(또는 n-gram 1개)**만 부호를 뒤집는다.
  - 붙임 표현(예: `안좋다`, `못한다`)은 unigram 직접 매칭이 실패했을 때 마지막으로 `안/못 + base` 형태를 시도해 부호를 뒤집는다.
