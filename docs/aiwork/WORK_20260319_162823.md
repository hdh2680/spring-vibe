# 목표 (goal)
프로젝트 실행 시 `YoutubeCommentService` 빈 생성이 실패하는 원인을 해결한다.

# 핵심 변경사항 (key changes)
- `DEFAULT_STOPWORDS`를 `Set.of(...)`로 만들 때 중복 요소("이") 때문에 `IllegalArgumentException: duplicate element`가 발생해 클래스 초기화가 실패했음을 확인했다.
- 중복이 있어도 안전하게 동작하도록 `Set.copyOf(new HashSet<>(List.of(...)))`로 변경했다.
- 편집 과정에서 유입된 UTF-8 BOM(\ufeff)을 제거해 컴파일이 정상 동작하도록 정리했다.

# 수정/추가한 파일 (touched files)
- `src/main/java/springVibe/dev/users/youtubeComment/service/YoutubeCommentService.java`

# 실행/테스트한 명령 (run/test commands)
- `.\mvnw.cmd -q -DskipTests compile`

# 후속 할 일 (follow-ups)
- PowerShell `Set-Content -Encoding utf8` 사용 시 BOM이 들어갈 수 있으니, Java 소스 편집은 UTF-8 no-BOM을 유지하는 방식으로 통일.
