# 작업 노트 (AI)

- 작성 시각: 2026-03-20 14:39:59 (Asia/Seoul)

## 목표 (goal)
- YouTube 댓글 분석 화면/검색 화면 UI를 사용하기 좋게 정리한다.
- 주제별 묶음(클러스터)에서 `other` 비중이 과도하게 커지는 원인을 개선하기 위해 “주제 키워드 묶음” 기반 군집 로직으로 변경한다.

## 핵심 변경사항 (key changes)
- 키워드 분석 화면(`youtubeCommentAnalysisView.html`)
  - Top10 키워드 vs 주제분포 비율을 5:5로 조정.
  - Top10 키워드 테이블이 가로 스크롤 생기지 않도록 반응형 테이블/줄바꿈 처리.
  - 주제별 묶음 카드 그리드를 3개/줄(1200px 이하 2개, 980px 이하 1개)로 변경.
  - 주제별 묶음에서 `Sample Comment IDs` 표시 제거.
  - 주제별 묶음의 대표키워드 표시를 “Top10 키워드처럼 테이블 형태(Rank/Term/Count/Bar)”로 변경.
  - 좁은 카드에서 테이블이 부모 영역을 넘지 않도록 `min-width`, bar 크기, table-layout, grid-item shrink 등을 조정.
  - 퍼센트(예: `100.0%`)가 줄바꿈되지 않도록 클러스터 테이블의 chart-row 레이아웃을 nowrap으로 고정.
  - 주제분포 도넛 차트를 더 크게 보이도록 responsive(`clamp`) 크기 조정.
- 이력 목록 화면(`youtubeCommentAnalysis.html`)
  - `Original Saved / Preprocessed Saved / Analysis Requested` 날짜 컬럼을 `yyyy-MM-dd HH:mm:ss` 형식으로 포맷 출력(Null은 `-`).
- 검색 화면(`youtubeCommentSearch.html`)
  - URL 입력/개수/버튼 레이아웃을 grid 기반으로 정리하고 버튼을 primary/secondary 톤으로 분리.
  - 패널 내부에 `panel-inner` 적용(기본 padding 확보).
  - 최초 진입 시에도 하단에 “빈 테이블”을 먼저 렌더링하도록 추가(검색 후에는 기존 결과 테이블이 표시됨).
- 주제별 묶음 알고리즘(`YoutubeCommentService`)
  - 기존: seed(대표 단어) 1개 포함 여부 기반 분류(`seed-keyword-overlap-v1`)로 인해 `other`가 크게 나올 수 있음.
  - 변경: seed를 “연관 키워드 묶음(seedKeywords)”으로 확장(댓글 단위 co-occurrence 기반)하고, 댓글-주제 점수를 overlap 점수로 계산해 할당(`seed-keyword-cooccurrence-v1`).
  - 결과 JSON에 `seedKeywordGroups`, 각 cluster의 `seedKeywords`를 포함해 튜닝/검증 가능하게 함.
