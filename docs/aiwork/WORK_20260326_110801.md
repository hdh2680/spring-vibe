# 작업 노트 (AI Work Log)

## goal
- DevSearch(개발 블로그 검색) 기능을 실제로 사용할 수 있도록 목록/상세 화면과 ES 기반 검색(ES -> pk 추출 -> DB hydrate) 및 재색인을 구현한다.

## key changes
- DB 목록/상세(Thymeleaf) 화면 추가:
  - `GET /users/devSearch/list`
  - `GET /users/devSearch/view?id=...`
  - `POST /users/devSearch/reindex` (ES enabled 시 DB -> ES 전체 재색인)
- Elasticsearch 문서 모델을 `velog_post`에 맞게 정리:
  - `VelogPostDocument` + `VelogPostDocumentRepository`
  - 부팅 시 ES 접속/인덱스 생성 방지를 위해 `@Document(createIndex=false)` 유지
- 검색 흐름 구현:
  - 검색어 있을 때 ES enabled면 ES에서 검색 -> hit id 순서대로 DB에서 `velog_post` 조회 후 정렬 유지
  - ES disabled 또는 ES down이면 DB LIKE 검색으로 fallback
- JPA 목록/검색 성능을 위해 `VelogPostSummary` projection 추가 및 `VelogPostRepository`에 page/search 메서드 추가
- `PRD-devSearch`의 검색서비스 단락을 실제 구현(화면/REST, 토글 동작) 기준으로 업데이트
