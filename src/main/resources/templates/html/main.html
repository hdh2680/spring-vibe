Spring 기반 Vibe Coding
유튜브 API로 데이터 수집·전처리·분석을 수행하고, 결과를 Ollama LLM으로 재확인(검토)합니다. 또한 Elasticsearch 기반 개발 블로그 검색(LLM 번역 검색어 포함)을 제공하며, 아이디어를 미니 프로젝트처럼 계속 확장할 예정입니다.
유튜브 댓글분석
수집 → 전처리 → 분석 → 감정분석 LLM 확인
수집
YouTube Data API
URL 입력 → 댓글 수집 → JSONL 저장
전처리
Normalize
정규화, 노이즈 제거, 중복/반복 축약
분석
Techniques
  1. 키워드 분석: Top 키워드, 토픽 그룹
  2. 네트워크 분석: 동시출현(Word Network)
  3. 감정 분석: 사전 기반 분류 + 추이
LLM 확인
Sentiment Review
감정분석 결과를 Ollama LLM으로 재확인(검토) 검토 결과를 커스텀 사전(Custom Lexicon) 업데이트로 반영
개발블로그검색 (Velog)
수집(별도 실행) → ES 색인 → 검색/조회
수집
Velog Ingest
별도 실행: springVibe.etc.velog.VelogIngestApplication 수집 결과를 DB(velog_post)에 적재
색인
ES Reindex
DB 글을 Elasticsearch로 색인 POST /users/devSearch/reindex 또는(ES enabled) POST /users/devSearch/es/reindex-all
조회
Search UI
페이지: /users/devSearch/list ES 사용 시: ES로 id/정렬 선택 → DB hydrate, ES 비활성/장애 시: DB LIKE fallback
Amazon Product
데이터 적재(약 140만건) → ES(영문) 색인 → 한글 검색(LLM 번역) 조회
적재
Data Load
제품/카테고리 데이터 적재 (약 140만건) DB: amazon_product, amazon_category
색인
ES Index (EN)
영문 제품명(title)을 Elasticsearch로 색인 POST /users/amazonProduct/reindex 또는 POST /users/amazonProduct/es/reindex-all
검색
Korean Query
한글 검색어 입력 시 Ollama LLM으로 영문 키워드 번역 후 ES 조회 동의어 사전 기반 쿼리 확장(추가 적용)으로 검색 품질 개선 페이지: /users/amazonProduct/list
