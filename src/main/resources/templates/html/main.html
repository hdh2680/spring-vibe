  2. 네트워크 분석: 동시출현(Word Network)
  3. 감정 분석: 사전 기반 분류 + 추이

Sentiment Review

기대효과
1. 댓글 파악 자동화
  • 대량 댓글을 빠르게 분석
  • 영상에 대한 사용자 반응 즉시 파악
의사결정 지원 데이터 기반 분석
2. 감정 분석 정확도 향상
  • 사전 기반 분석의 한계(반어법, 문맥 문제) 보완
  • LLM을 활용한 2차 검증으로 오류 감소
하이브리드 분석 구조 정확도 개선
3. 비용 효율적인 AI 활용
  • 전체 댓글을 LLM으로 처리하지 않음
  • 불확실한 댓글만 선택적으로 재검증
비용 최적화 선별적 LLM 사용
4. 핵심 키워드 및 관계 분석
  • 주요 키워드 추출로 관심사 파악
  • 키워드 간 관계로 이슈 구조 분석
트렌드 분석 관계 기반 인사이트
5. 시각화 기반 인사이트 제공
  • 감정 비율, 키워드 랭킹, 네트워크 그래프
  • 사용자 친화적 분석 화면 제공
데이터 시각화 사용자 친화적 분석
개발블로그검색 (Velog)
수집(별도 실행) → ES 색인 → 검색/조회
수집
Velog 수집
별도 실행: springVibe.etc.velog.VelogIngestApplication 수집 결과를 DB(velog_post)에 적재
색인
ES 색인
DB 글을 Elasticsearch로 색인 POST /users/devSearch/reindex 또는(ES enabled) POST /users/devSearch/es/reindex-all
조회
검색 UI
페이지: /users/devSearch/list ES 사용 시: ES로 id/정렬 선택 → DB hydrate, ES 비활성/장애 시: DB LIKE fallback
Amazon Product
데이터 적재(약 140만건) → ES(영문) 색인 → 한글 검색(LLM 번역) 조회
적재
데이터 적재
제품/카테고리 데이터 적재 (약 140만건) DB: amazon_product, amazon_category
색인
ES 영문 색인
영문 제품명(title)을 Elasticsearch로 색인 POST /users/amazonProduct/reindex 또는 POST /users/amazonProduct/es/reindex-all
검색
한글 검색
한글 검색어 입력 시 Ollama LLM으로 영문 키워드 번역 후 ES 조회 Ollama LLM 기반 키워드/동의어 생성으로 쿼리 확장(추가 적용)하여 검색 품질 개선 LLM 번역 및 동의어 품질 향상을 위해 지시문(prompt)을 강화 Redis 캐싱으로 동일 검색어의 LLM 결과를 재사용(추가 적용) 페이지: /users/amazonProduct/list
기대효과
1. 대용량 데이터 기반 검색 시스템 구축
  • 140만 건 상품 데이터 적재 및 검색
  • ES를 활용한 빠른 검색 처리
대용량 데이터 처리 검색 시스템 설계
2. 다국어 검색 문제 해결
  • 영어 데이터 기반 상품을 한국어로 검색 가능
  • LLM을 활용한 번역 + 동의어 확장
검색 정확도 향상 사용자 경험 개선
3. 비용 효율적인 LLM 활용
  • 전체 데이터 사전 번역을 하지 않음(비용 과다)
  • 검색 시점에만 LLM 호출
비용 절감 시스템 유연성 확보
4. Redis 캐싱을 통한 성능 개선
  • 동일 검색어에 대해 LLM 결과를 Redis에 캐싱(추가 적용)
  • 반복 요청 시 LLM 호출 없이 즉시 응답
응답 속도 대폭 개선 서버 부하 감소 LLM 호출 비용 절감
